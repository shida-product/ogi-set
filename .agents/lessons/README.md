# Lessons カテゴリ別アーカイブ

`.agents/lessons.md`（必読サマリ）に対する **詳細ログのオンデマンド参照先**。
毎セッションで全カテゴリを読む必要はない。着手ドメインに応じて該当カテゴリだけ読み込む。

---

## カテゴリ一覧

| ファイル                   | 着手前に読むタイミング                 |
| -------------------------- | -------------------------------------- |
| [encoding.md](encoding.md) | ターミナルで日本語ファイル操作を行う前 |
| [security.md](security.md) | 認証・API キー・秘密情報を扱う前       |

プロジェクトが進むにつれ、カテゴリを追加してください（例: `python.md`, `web.md`）。

---

## 積み上げ学習プロトコル

### 1. 追記（Capture）

新しい教訓は **最も該当するカテゴリ 1 ファイルにだけ** 追記する。

### 2. タグ付け（Indexing）

タイトル先頭に `[Tag1][Tag2]` 形式でタグを付ける。

### 3. 統合（Master Rule 化）

類似教訓が 3 件以上溜まったら、ファイル冒頭の「📌 Master Rules」に集約原則を昇格させる。

### 4. サマリ昇格（Critical 化）

本番事故・再発 3 回以上・横断 Audit で重要と判明した教訓は `.agents/lessons.md` へ格上げする。

### 5. 退役（Archive）

設計変更で永久に当てはまらなくなった場合、本文末尾に `> 🗄️ Archived: YYYY-MM-DD 理由` を追記する。削除はしない。

---

## 追記時のフォーマット

```markdown
### YYYY-MM-DD [Tag1][Tag2] 教訓のタイトル

- **❌ Anti-pattern:**
  本文…

- **✅ Solution / Rule:**
  本文…
```

## 確定仕様・境界

- 本体はフロントエンド単一ファイル（HTML+CSS+vanilla JS）。ビルド工程なし.
- Python は未導入。pre-commit の文字化けlintは Python 導入までは自動 skip される。
- 整形/Lint は npm（prettier / eslint）。使う場合は `npm install` を先に実行。
- 既存の CSS クラス名・価格計算ロジックは無断で改変しない（モック整合）。
- **薬リストはグループ分けなしのフラット表示**。`group` フィールド・`groupOrder()`・`.group` / `.group-label` CSS は撤去済み（GAS版と見た目統一）。
- トラネキサム酸250mg×2のラベル文言は「用意する」で統一。
- 注釈およびフッターの「仕組み」は読みやすさ向上のため適切に改行表示する。
- 数量増減ボタン（.qbtn）はタップしやすさ向上のため幅42px・高さ42px（約1.3倍）とする。
- 「トラネキサム酸を 250mg×2 で用意する」のオプション（`.opt-row`）は、「トラネ500mg「YD」」の行の直下にマージン（`margin-left: 15px`）を適用して動的に挿入配置する。
- 上記オプションがONの時、「トラネ500mg「YD」」を含むセット（2種および3種）は一律 +¥500 が適用され、領収書に注記が表示される。
- ただし、2種セットの「トラネキサム酸＋トコフェロール」および「トラネキサム酸＋ユベラ200mg」は適用外（除外）のまま維持する。
- 会計パネル（`.result`）全体を画面下部に固定表示（スティッキー化）する。薬が選ばれているときのみフワッと表示され、内訳の行数に応じてパネルが自動的に上方向に拡張し、すべての内訳と合計がスクロールなしで一覧表示される構造とする。
- 会計パネルの動的拡張に追従して「ノイロビタン配合錠」やフッターが隠れないよう、パネルの実際の高さ（`offsetHeight`）をJSで動的に検知し、全体のパディング（`padding-bottom`）を自動調整（パネル高さ + 30px）する。
- オプションの注釈文（`.note`）内に対象となる2種セットおよび3種セットの具体的な構成薬名を明記して分かりやすくする。
- GitHubリモートリポジトリ（shida-product/ogi-set）と紐付け、オンライン上でバージョン管理を開始。
- **運用は GAS（スプレッドシート起動ダイアログ）一本化**。GitHub Pages / スタンドアロン版（`set-price-optimizer.html`）は廃止・削除済み（`.claspignore` の該当行も除去）。UI 本体は `index.html`。
- 1つのシート「マスタ」から縦並び（セルの結合なし）のレイアウトで商品（グループ列は除外）と価格・セット定義をロードする GAS スクリプト (`Code.js`) を実装。A1:商品名、A15:2種セット、A33:3種セット、A45:4種目以降の追加、というセル内の見出し文字列を直接検知してセクションを切り替える設計。この「マスタ」シートと初期データを結合なしで全自動生成するセットアップ機能 (`setupMasterSheet`) もメニューに搭載。
- **価格の正本はスプレッドシート「マスタ」**。GAS起動時に `getMasterData()` が `DATA` を上書きするため、シートが常に勝つ。改定は年1回・シートのみ触る運用（`DEFAULTS` / `setupMasterSheet` の年次同期は不要）。
- `index.html` の `DEFAULTS` は**シート読込失敗時の非常用フォールバックのみ**（コメントも修正済み。旧「唯一の正本」表記は撤回）。画面上部の「再読込」ボタンで再ロード可。clasp接続済みで `npx clasp push` により直接GASへ反映可能。
- **eslint を実効化**：`eslint.config.js` に GAS（`SpreadsheetApp`/`HtmlService`/`google`）・ブラウザ・Node の globals を定義し、Code.js のGASグローバル誤検知を解消。品質ゲートは現状エラー0（残るのは既存hookの空catch警告のみ）。
- 会計パネルの「単品ばらし」表記は「単品価格」に変更。
- 3種セットにおけるトラネ500mg「YD」を含むセット定義について、トラネ250mg「YD」が選択されている場合も同じものとして代替して3種セットが適用（セット化）できるように適用判定を拡張（2種セットでの代替は対象外）。

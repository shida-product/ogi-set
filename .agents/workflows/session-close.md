# セッションクローズ（自律実行）

ユーザーから「書き換えて」と指示されなくても、**作業単位が一区切りついたら** 次を自主的に実行する。

## 実行タイミング

次のいずれかに該当したらクローズ手順を走らせる:

- 依頼されたタスクが完了した
- 一区切りの機能・修正・調査が完了した
- セッションを終える直前
- 10 ステップ超の連続作業の区切り（A3 に従い一旦停止した後）

## 手順（順番固定）

### 0. `.agents/state/locks.md` から自分の行を削除（必須）

- このセッション ID で追記した行をすべて削除する。
- 30 分以上更新のない他セッションの行があれば stale として削除してよい。

### 1. 品質ゲート

- Python: `ruff format .` → `ruff check --fix .`
- Web（あれば）: `npm run format` → `npm run lint`
- 通らないものは完了報告しない。修正するか理由を報告する。

### 2. `.agents/handover.md` を更新（必須）

更新する項目:

| 項目           | 書く内容                         |
| -------------- | -------------------------------- |
| Current Focus  | 今の作業焦点（1〜3 行）          |
| Next Actions   | 未完了タスクと状態（☐ / ◐ / ✅） |
| 確定仕様・境界 | 今回確定した制約・触らない領域   |

**書かないもの**: 完了済みの詳細ログ（→ changelog へ）、再発防止の教訓（→ lessons へ）

handover が 150 行超なら、古い完了詳細を changelog に移してから保存する。

### 3. `.agents/lessons/` を更新（該当時のみ）

次に該当したら **ユーザー指示なしで** 追記する:

- 同じミスを再発させそうなパターンを見つけた
- ユーザーから修正・指摘を受けた
- 品質ゲートやツールで想定外の失敗があった

追記先は `.agents/lessons/<category>.md` 1 ファイル。プロトコルは `lessons/README.md` に従う。
Critical に値するものは `.agents/lessons.md` への昇格を検討する。

### 4. `.agents/changelog.md` を更新（該当時のみ）

次に該当したら追記する:

- マイルストーン級の完了（機能追加、設計判断の確定など）
- 後から「なぜそうしたか」を追いたくなる変更

1 行要約 + 判断理由程度に留める。ファイル単位の差分は Git に任せる。

### 5. Git コミット（該当時・バッチ）

コミット方針は `.agents/workflows/git-safety.md` §コミットのタイミング に従う。

- **関心事ごとに分割**する（1 コミット = 1 関心事）
- **逐一コミットしない** — その関心事が固まってから 1 コミット
- 複数の関心事が完了していれば、**複数コミットに分けてまとめて** 実行してよい
- `git push` はユーザー明示指示時のみ

## 完了報告に含めること

ユーザーへの報告では次を簡潔に触れる:

- locks を解除したか
- handover を更新したか
- lessons / changelog を更新したか（更新した場合は何を）
- コミットしたか（した場合はメッセージ一覧）

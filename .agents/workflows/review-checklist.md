# レビューチェックリスト（人間 = 監査役）

AI がコーディングを完了したら、以下を目視で確認する。

## 1. エラー・警告（Error Lens）

- [ ] 赤いエラーテキストが残っていないか
- [ ] 警告が意図的なものか（不要な警告は AI に修正依頼）

## 2. 差分の監査（GitLens）

- [ ] AI が変更したファイル一覧を確認したか
- [ ] 意図しない破壊的変更（削除・リネーム）がないか
- [ ] 指示外の広範囲リファクタや整形差分が混入していないか

## 3. 動作確認

- [ ] 変更範囲に応じた最小限の動作確認をしたか
- [ ] AI が「テスト未実行」と報告している場合、代替確認をしたか

## 4. セキュリティ

- [ ] 秘密情報がコードやコミットに含まれていないか
- [ ] 外部入力の検証・サニタイズが適切か（該当する場合）

## 5. 引き継ぎ

- [ ] `.agents/handover.md` が現状を反映しているか
- [ ] 次セッションで迷わないレベルの情報が残っているか

# AI セッション運用

## 基本方針

- コーディングは AI が主導する。人間は **レビューと目視チェック（監査役）** に徹する。
- **どの AI / エディタを使うかはユーザーが選ぶ。AI 側でツールの割当てや明示指示はしない。**
- どの AI でも同じ `.agents/` ルールと handover に従う。

## セッション開始時（自律）

ユーザーから「handover を読め」と言われなくても、作業前に次を読む:

1. `.agents/RULES.md`
2. `.agents/handover.md`
3. `.agents/state/locks.md`
4. `.agents/lessons.md`
5. タスクに応じた `.agents/workflows/*.md`

## ロックの取得（編集開始前・自律）

1. `locks.md` で他セッションのロックを確認する。
2. 触る予定のファイルがロックされていなければ、表に自分の行を追記する。
3. ロック済みファイルが必要なら:
   - **別ファイルだけ**なら並列で作業を続ける
   - **そのファイルが必須**なら handover の Next Actions を更新し、別タスクに切り替えるかユーザーに報告する
4. **並列作業は禁止しない**。ロックされていないファイルなら同時に編集してよい。

## 複数 AI / エディタ併用時の競合回避

ツールの指定はしない。`locks.md` で衝突だけ避ける:

1. ロック中のファイルは編集しない。
2. ロック外のファイルは並列編集してよい。
3. 共有ファイル（handover、ルール類）は必要時のみ更新。衝突したら先の内容を尊重して統合する。

## Format on Save ループ対策

AI が「書き込みに失敗した」と繰り返す場合:

1. `editor.formatOnSave` を一時的に `false` にする。
2. 書き込み完了後に `ruff format` / `npm run format` を手動実行する。
3. 原因が解消したら `formatOnSave` を再有効化する。

## セッションクローズ（自律）

作業の区切りでは **ユーザー指示なしで** `.agents/workflows/session-close.md` を実行する。

# Git 安全ルール

## 禁止（ユーザー明示許可なし）

- `git reset --hard`
- `git push --force`（特に `main` / `master`）
- 履歴改変（rebase -i 等の対話的操作）
- 広範囲のファイル削除
- `git push`（コミットはしてよいが、push は指示があるまで行わない）

## コミット規約

- Conventional Commits 準拠: `feat:` / `fix:` / `refactor:` / `docs:` / `style:` / `test:` / `chore:`
- メッセージは日本語で「なぜ」を書く。
- **1 コミット = 1 関心事**（内容で分割する）

## コミットのタイミング（バッチ方針）

| やること                                                                      | やらないこと                      |
| ----------------------------------------------------------------------------- | --------------------------------- |
| 1 つの関心事が**固まったら** 1 コミット                                       | ファイル保存のたびにコミット      |
| セッション内で複数関心事が完了したら、**関心事ごとに分けて** まとめてコミット | 無関係な変更を 1 コミットに混ぜる |
| docs だけ / code だけ / chore だけは分ける                                    | 「とりあえず全部」1 コミット      |

### 関心事の例

```
feat: ユーザー認証のログイン処理を追加     ← 認証機能が固まった
fix: ログイン失敗時のエラーメッセージを修正  ← 別のバグ修正（別コミット）
docs: handover と lessons を今回分更新      ← .agents 更新だけ（別コミット）
```

### コミット前チェック

- [ ] このコミットは 1 つの関心事だけか
- [ ] 秘密情報（`.env` の値等）が含まれていないか
- [ ] 品質ゲート（Ruff / ESLint）を通したか
- [ ] 不要なデバッグ出力・コメントアウト塊を残していないか

## 未コミット変更の扱い

- 他セッションや人間の変更として扱い、勝手に巻き戻さない。
- 変更の意図が不明な場合はユーザーに確認する。
